## Follow-ups

- **E2E via zero-native automation hooks** ‚Äî needs upfront research into the automation surface (`-Dautomation=true` flag exists in `build.zig:40`, but the driver API isn't documented in the api-lab CLAUDE.md). Filed as a separate P2 backlog file so the prerequisite (research + driver) is properly scoped: `docs/backlog/P2-<ts>-e2e-zero-native-automation.md`.
- **Cold-start benchmarking** ‚Äî defer until first-paint harness exists. Cold-start is currently <200 ms on the dev box; no immediate regression risk.
- **Cross-platform release builds (Linux + Windows)** ‚Äî `release.yml` ships macOS arm64 + x86_64 in v1. Adding Linux/Windows requires cross-compile work and integration with the `zero-native package` CLI. Will need its own slice once those moving parts settle.
- **`build.zig` host-`npm` calls** ‚Äî `frontend-install` (line 99) and `frontend-build` (line 103) shell out to host `npm`, which violates the `frontend/CLAUDE.md` dnpm-only policy. Should route through `dnpm` or be removed (the `dnpm run build` flow is the documented path; these zig steps are unused in practice).
- **`default_zero_native_path = "../zero-native"` breaks under worktrees** ‚Äî the worktree path is one level deeper than primary, so `../zero-native` resolves to a sibling that doesn't exist. Either auto-discover via `git rev-parse --show-toplevel`'s sibling, or document the per-worktree symlink workaround.
