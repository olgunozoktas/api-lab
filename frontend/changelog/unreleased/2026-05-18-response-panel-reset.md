---
title: Selecting a request now refreshes the response panel
date: 2026-05-18
---

Picking a saved request from the Collections sidebar (or a History
entry) used to leave the response panel showing the **previous**
request's response — and stuck on whatever tab was open, like
Visualize or Headers.

Now selecting any request clears that stale response and returns the
panel to the **Body** tab. A request you haven't sent shows the empty
response state, as it should — no more mismatched data from another
request.
