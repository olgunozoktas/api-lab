API Lab — a native macOS API tester (Postman-style) built on the
zero-native Zig + WebKit shell. CORS-free HTTP via curl subprocess,
WebSocket / SSE / gRPC tabs, request history, environments,
collections, GraphQL composer, multi-tab workspace, TLS config,
Postman import, OAuth helper, pre/post-request scripts, examples,
themes, in-app guides + changelog.

This entry backfills the major features that landed during the v0.1.0
build-up — every meaningful slice from the very first commit through
the changelog modal that surfaces this list. From v0.1.1 forward,
each user-visible change ships as its own `frontend/changelog/unreleased/<slug>.md`.

## Architecture + foundation

- **Native macOS shell on zero-native** — Zig 0.16 binary hosting a
  WKWebView (Apple's system web engine, NOT Chromium). The app window
  uses `WebViewSource.assets()` so the React frontend serves under the
  `zero://app` origin — required for `localStorage`, `Headers`, IDB,
  and other browser APIs that misbehave under null-origin loads.
- **JS↔Zig bridge** — `window.zero.invoke(cmd, payload)` round-trips
  through a policy check (origin + permission) into a Zig handler.
  Currently exposes one command: `http.request` (curl subprocess for
  CORS-free HTTP). 1 MB per-call output cap.
- **React + Vite + TypeScript + Tailwind v4 frontend** — Phase A
  rewrite from the original vanilla HTML/JS prototype. Tailwind v4 with
  `@theme` tokens, `@vitejs/plugin-react`, source maps in production
  builds for ErrorBoundary stack-frame resolution.
- **shadcn/ui primitives** — Button, Dialog, Select, Tabs,
  ContextMenu, SearchInput, Tooltip — all `forwardRef`-friendly and
  composition-friendly (Radix `Slot` patterns where it fits).

## Composer + protocols

- **HTTP composer** — method picker, URL bar with `{{var}}`
  substitution, Params / Headers / Auth / Body / Scripts / GraphQL
  sub-tabs. Body modes: none / json / form-urlencoded / raw with a
  CodeMirror 6 editor for JSON + raw.
- **GraphQL composer** — auto-detected when the URL points at a
  GraphQL endpoint; method becomes POST, body serialized as
  `application/json`. Query + variables in dedicated panes; reuses
  the body-editor highlighter.
- **WebSocket panel** — `ws://` and `wss://` URLs activate the WS
  tab. Connect / disconnect, send messages with `⌘ Enter`, ping
  button, per-connection log with auto JSON detection + JSON tag.
  Status pill cycles through BEKLİYOR / BAĞLANIYOR / AÇIK / KAPANIYOR
  / KAPALI / HATA.
- **Server-Sent Events panel** — `sse://` and `sses://` URLs activate
  the SSE tab. Auto-reconnect handling, named-event tags, last-
  event-id cursor capture, manual force-reconnect.
- **gRPC panel** — `grpc://` and `grpcs://` URLs activate the gRPC
  tab. Routes through `grpcurl` (must be installed; brew install
  hint surfaces inline if missing). Unary + server-streaming via
  batched-collect approach.
- **gRPC reflection service browser** — when the server has reflection
  enabled, list services + methods in a sidebar; pick a method to
  generate a request-message skeleton from the proto schema. Per-target
  5-min cache with `(önbellek, Xs önce)` badge on hits; manual
  **Yenile / Refresh** invalidates.
- **gRPC TLS / mTLS** — paste-in CA cert, client cert, client key
  (PEM) with per-call temp files. Hardened permissions: `0o700` for
  the dir, `0o600` for each PEM file (cleaned up on call completion).
  SNI override + authority override supported. Per-call dir under
  `/tmp/api-lab-grpc-<hex>/`.
- **OAuth 2.0 helper** — paste-in access token + Bearer auto-injection.
  Refresh button hits `token_url` with `refresh_token` + `client_id`
  to mint a new token without leaving the app. PKCE + popup flow is
  documented as a follow-up (zero-native upstream changes needed).

## Workspace + organization

- **Collections sidebar** — folder-tree with inline rename
  (double-click), drag-to-move, recursive delete (folder takes its
  kids), right-click context menu (new sub-folder / new request /
  rename / delete), and live search filter across visible nodes.
- **History sidebar** — every request lands here with method + status +
  size + elapsed-ms badges. Smart search across method + status + URL
  substring (e.g. `GET 401 login` finds 401s on a login endpoint).
  Clear button confirms before purging.
- **Multi-tab workspace** — `⌘ T` new, `⌘ W` close (last tab is replaced
  with a fresh empty one rather than vanishing), `⌘ 1`–`⌘ 9` jump.
  Each tab carries its own request body / response state /
  composer-tab / response-tab; collections / environments / history
  are shared.
- **Quick switcher** — `⌘ P` opens a fuzzy palette across open tabs +
  saved collections + recent history. Smart substring scorer with
  tab-prioritization. `↑`/`↓` to navigate, `↵` to open, `Esc` to
  dismiss.
- **Postman v2.1 import** — drag-drop a collection JSON to bring the
  whole tree + collection variables over in one shot. Imports land
  under a wrapper folder named after the source so multiple imports
  don't pollute the root. GraphQL requests detected and routed to
  the GraphQL composer.
- **Resizable 3-pane layout** — drag the dividers between Sidebar /
  Composer / Response. Double-click a divider to reset. Widths
  persist across reloads via the same Zustand persist middleware
  that backs collections + history.

## Authoring + automation

- **Environments** — multiple named contexts, each with its own KV
  var set. Active env in the top-bar dropdown; `{{var}}` substitution
  happens at send-time across URL / headers / params / body. The
  active env's `vars` is exposed as a stable selector so React
  components don't re-render on unrelated state changes.
- **Save as variable** — right-click any value in the response viewer
  to save it into a named env variable (`{{access_token}}`, etc.) for
  use in later requests. Surface lives next to the **Copy as code**
  menu.
- **Examples** — captured response snapshots attached to a saved
  request. Save / view / rename / delete from the Examples tab. Persist
  with the request so they survive reloads. Mock-server (Zig sidecar
  serving examples over HTTP) is queued as a follow-up.
- **Pre / post-request scripts** — QuickJS sandbox with a `pm.*` API
  subset (`pm.test`, `pm.expect`, `pm.environment.set`, `pm.response`,
  etc.). Pre-scripts mutate the request before send; post-scripts
  run on the response. Scripts panel surfaces test pass/fail tallies
  + console output inline. Sandbox limits: 5 s CPU, 10 MB heap, no
  `fetch` / `XHR`.
- **Copy as code** — top-of-response dropdown with 6 formatters
  (curl, fetch, axios, Python requests, Go net/http, JS XMLHttpRequest).
  Renders the active request fully resolved (env vars expanded,
  auth applied) — paste-ready.
- **cURL paste-detect / import** — paste a curl command into the URL
  bar and a banner offers to parse + populate the composer. Handles
  `-X`, `-H`, `-d`, `--data`, `-u`, `--insecure`, `-L`, etc.

## Cancellation + observability

- **Request cancellation** — Send button morphs into a red `⨯ İptal /
  Cancel` while in flight. `⌘ +.` (canonical macOS abort) fires the
  same handler. AbortSignal threaded through the fetch path; native
  bridge path returns UI to ready immediately, with documented
  soft-cancel semantics for the underlying curl subprocess.
- **Aria-live response status** — screen readers announce status
  code changes when a request lands; high-contrast theme available
  for visual alternative.
- **Astro-style error overlay** — when the React tree crashes, an
  in-app overlay surfaces a syntax-highlighted blame frame, parsed
  component stack, source excerpts, and a one-click report-copy.
  Production source maps resolve at runtime so paste-able stacks
  reference real function + file names instead of minified ids.

## Look + feel

- **Themes** — system / light / dark / Tokyo Night / GitHub Light /
  high-contrast. Auto follows the OS appearance unless overridden.
  Theme picker lives in **Ayarlar / Settings** (top-bar gear).
- **Settings modal** — central place for theme, language, default
  request behavior (timeout-ms, max-redirects, skip-TLS), and a
  full keyboard shortcut reference.
- **Turkish + English i18n** — every user-facing string flows through
  `t()`. TR is the source of truth (`frontend/src/lib/i18n/tr.ts`);
  EN mirrors it exactly. TypeScript fails the build if any locale
  is missing a key. Adding a new language is a single dictionary
  file plus three lines in `i18n/index.ts`.
- **In-app feature guides** — `?` shortcut or top-bar help-circle
  button opens a modal with curated walkthroughs (Quick start,
  Environments, gRPC, Request cancellation, Quick switcher,
  Examples, Streaming protocols, Postman import). Live search across
  title / group / body. Markdown bodies bundled at build time via
  `import.meta.glob`.
- **In-app changelog** — top-bar clock-history button (or the
  auto-trigger on first launch after a version bump) shows this
  timeline as a vertical list of cards. The lib/markdown.ts subset
  renderer is escape-by-default safe; no marked / dompurify
  dependency in the bundle.
- **Toast notifications** — non-modal feedback for save / cancel /
  copy / theme switch. Auto-dismiss; stacked.

## Reliability + plumbing

- **IndexedDB persistence** — collections, environments, history,
  tabs, and preferences live in IDB. Uncaps the old localStorage
  5 MB limit (multi-MB Postman imports + OpenAPI specs no longer
  hit `QuotaExceededError`). One-shot transparent migration on
  first read for users upgrading from the localStorage-era
  snapshot. Persist contract is versioned with `migrate(persisted,
  fromVersion)` callbacks (v1 → v2 → v3 chain).
- **Native dialogs replaced** — `window.confirm` / `alert` /
  `prompt` are silent no-ops in WKWebView under custom origins; we
  swap them with React modals (Dialog + ConfirmDialog + PromptDialog)
  driven by a `useConfirm()` hook so any code path that needs
  confirmation gets a real surface.
- **+ New folder / + New request inline-rename** — sidebar create
  flows replaced the broken `prompt()` path with auto-named drafts
  + inline rename mode. Press `↵` to accept, `Esc` to cancel.
- **TreeNode cycle guard** — drag-into-folder validation refuses to
  drop a folder into one of its own descendants (would create a
  cycle in the collection tree).
- **React #185 hardening** — selectors that returned a fresh empty
  object/array each render were stabilized to module-scope
  `Object.freeze({})` constants behind a `??` fallback. Two
  separate occurrences fixed (QuickSwitcher, ExamplesPanel).
- **Bundle-size guardrail** — `scripts/check-bundle-size.sh` enforces
  1300 KB JS raw / 400 KB gzipped in CI. Current bundle ~1207 raw
  / 367 gz; warning threshold raised to 1400 to match.
- **Tests + CI** — Phase F: Zig unit tests for the http handler
  (argv composition, header parsing, JSON encoding, error
  surfacing); Vitest for pure utils + store slice actions + lib
  primitives; GitHub Actions running both on every push.
- **Pre-commit hooks** — `zig fmt` + `prettier --check` via dnpm.
  Idempotent install via `bash scripts/install-hooks.sh`.

## Build + supply chain

- **`./build.sh`** — one-shot frontend + Zig build that kills any
  prior instance, wipes the WebKit asset cache, and launches the
  freshly-built binary. Flags for release-mode, no-run, frontend-
  only, zig-only, force-reset-state.
- **dnpm secure docker npm** — every npm/Vite operation runs inside
  a hardened OrbStack container: non-root UID, all caps dropped,
  custom seccomp profile, read-only rootfs + project mount, tmpfs
  with `noexec`, pinned npm registry, two-phase install (download
  scripts-off, rebuild offline), and auto-audit after every install.
  No Docker socket mounted (no escape vector). Malicious postinstall
  scripts can't reach `~/.ssh`, `~/.aws`, or browser profiles.
- **Project-rooted compose** — `docker-compose.yml` at repo root
  for clone-and-run convenience; `docker-compose.node.yml` (the
  hardened one) is gitignored and emitted by `dnpm setup`.

## Platform notes

- **macOS only** for v0.1.0 (`.platforms = .{ "macos" }` in
  `app.zon`). Linux / Windows builds are queued as follow-ups —
  zero-native's WebKit hosting needs platform-specific glue.
- **WebKit, not Chromium.** This is Apple's `WKWebView` (the same
  engine Safari uses), via zero-native's macOS host. Behaviorally
  that means stricter `localStorage` / `Headers` semantics than
  Chromium, no Chrome devtools / extensions, and a small set of
  WKWebView-specific gotchas (documented in `CLAUDE.md`).
