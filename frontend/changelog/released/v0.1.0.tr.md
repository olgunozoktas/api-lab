API Lab — zero-native Zig + WebKit shell üzerine kurulu native
macOS API tester (Postman tarzı). curl subprocess ile CORS-free
HTTP, WebSocket / SSE / gRPC sekmeleri, istek geçmişi,
environment'lar, koleksiyonlar, GraphQL composer, multi-tab
workspace, TLS config, Postman import, OAuth helper, pre/post-request
script'ler, examples, tema'lar, in-app rehber + changelog.

Bu entry v0.1.0 build-up sürecinde inen büyük özellikleri backfill
eder — ilk commit'ten changelog modal'ına kadar her anlamlı slice.
v0.1.1'den itibaren, her kullanıcı görünür değişiklik kendi
`frontend/changelog/unreleased/<slug>.md` dosyası olarak ship olur.

## Mimari + temel

- **zero-native üzerinde native macOS shell** — Zig 0.16 binary,
  WKWebView host'lar (Apple'ın sistem web engine'i, Chromium DEĞİL).
  Uygulama penceresi `WebViewSource.assets()` kullanır, böylece
  React frontend `zero://app` origin altında servis edilir —
  `localStorage`, `Headers`, IDB ve null-origin'de bozulan diğer
  browser API'leri için gerekli.
- **JS↔Zig bridge** — `window.zero.invoke(cmd, payload)` policy
  check'ten (origin + permission) Zig handler'a round-trip yapar.
  Şu an tek komut expose: `http.request` (CORS-free HTTP için curl
  subprocess). Çağrı başına 1 MB output cap.
- **React + Vite + TypeScript + Tailwind v4 frontend** — Phase A
  orijinal vanilla HTML/JS prototype'tan rewrite. `@theme` token'lı
  Tailwind v4, `@vitejs/plugin-react`, ErrorBoundary stack-frame
  resolution için production source map.
- **shadcn/ui primitive'leri** — Button, Dialog, Select, Tabs,
  ContextMenu, SearchInput, Tooltip — hepsi `forwardRef`-friendly
  ve composition-friendly (uygun yerde Radix `Slot` pattern'leri).

## Composer + protokoller

- **HTTP composer** — method picker, `{{var}}` substitution'lı URL
  bar, Params / Headers / Auth / Body / Scripts / GraphQL
  alt-sekmeler. Body modları: yok / json / form-urlencoded / raw,
  JSON + raw için CodeMirror 6 editör.
- **GraphQL composer** — URL GraphQL endpoint'e işaret ettiğinde
  otomatik algılanır; method POST olur, body `application/json`
  olarak serialize edilir. Query + variable'lar dedicated
  panellerde; body-editor highlighter'ı reuse edilir.
- **WebSocket panel** — `ws://` ve `wss://` URL'ler WS sekmesini
  aktif eder. Connect / disconnect, `⌘ Enter` ile mesaj gönder,
  ping butonu, otomatik JSON algılama + JSON tag'li bağlantı-başına
  log. Status pill BEKLİYOR / BAĞLANIYOR / AÇIK / KAPANIYOR /
  KAPALI / HATA döngüsü.
- **Server-Sent Events panel** — `sse://` ve `sses://` URL'ler SSE
  sekmesini aktif eder. Otomatik reconnect, named-event tag'leri,
  last-event-id cursor capture, manuel force-reconnect.
- **gRPC panel** — `grpc://` ve `grpcs://` URL'ler gRPC sekmesini
  aktif eder. `grpcurl` üzerinden geçer (kurulu olmalı; eksikse
  brew install hint'i sekmede inline görünür). Unary +
  server-streaming via batched-collect yaklaşımı.
- **gRPC reflection servis tarayıcı** — sunucuda reflection açıksa,
  servis + method'ları yan menüde listele; bir method seç, proto
  şemasından request-message iskeleti üret. Hedef başına 5-dk
  cache + hit'lerde `(önbellek, Xs önce)` rozeti; manuel **Yenile**
  invalidate eder.
- **gRPC TLS / mTLS** — paste-in CA cert, client cert, client key
  (PEM), çağrı başına temp dosya. Hardened permissions: dir için
  `0o700`, her PEM için `0o600` (çağrı tamamlanınca temizlenir).
  SNI override + authority override desteklenir. Çağrı başına dir
  `/tmp/api-lab-grpc-<hex>/`.
- **OAuth 2.0 helper** — paste-in access token + Bearer otomatik
  inject. Yenile butonu `token_url`'e `refresh_token` + `client_id`
  ile vurarak uygulama dışına çıkmadan yeni token alır. PKCE +
  popup flow follow-up olarak dökümante (zero-native upstream
  değişiklikleri gerek).

## Çalışma alanı + organizasyon

- **Koleksiyonlar yan menü** — folder-tree, inline rename
  (çift-tıkla), drag-to-move, recursive delete (klasör çocuklarını
  alır), sağ tık context menu (yeni alt klasör / yeni istek /
  yeniden adlandır / sil) ve görünen node'larda canlı arama filtresi.
- **Geçmiş yan menü** — her istek burada method + status + size +
  elapsed-ms rozetiyle iner. method + status + URL substring
  arasında akıllı arama (örn. `GET 401 login` login endpoint'inde
  401'leri bulur). Clear butonu purge öncesi confirm eder.
- **Multi-tab workspace** — `⌘ T` yeni, `⌘ W` kapat (son sekme
  kaybolmak yerine taze boşla replace edilir), `⌘ 1`–`⌘ 9` atla.
  Her sekme kendi request body / response state /
  composer-tab / response-tab taşır; koleksiyonlar / environment'lar
  / geçmiş paylaşılır.
- **Hızlı geçiş** — `⌘ P` açık sekmeler + kayıtlı koleksiyonlar +
  yakın geçmiş arasında fuzzy palette açar. Sekme öncelikli akıllı
  substring scorer. `↑`/`↓` gez, `↵` aç, `Esc` kapat.
- **Postman v2.1 import** — koleksiyon JSON'unu sürükle-bırak,
  tüm ağaç + koleksiyon değişkenleri tek seferde gelir. Import'lar
  kaynak adıyla wrapper folder altına iner, böylece birden fazla
  import root'u kirletmez. GraphQL istekleri algılanır ve GraphQL
  composer'a yönlendirilir.
- **Yeniden boyutlandırılabilir 3-pano layout** — Sidebar / Composer
  / Yanıt arası bölücüleri sürükle. Çift-tık reset. Genişlikler
  reload arası persist olur (koleksiyonları + geçmişi destekleyen
  aynı Zustand persist middleware).

## Yazma + otomasyon

- **Environment'lar** — birden çok adlı context, her biri kendi
  KV var setiyle. Aktif env üst bardaki dropdown'da; `{{var}}`
  substitution gönderme anında URL / header / params / body
  üzerinde çalışır. Aktif env'in `vars`'ı stable selector olarak
  expose, böylece React component'ler ilgisiz state değişikliklerinde
  re-render olmaz.
- **Değişken olarak kaydet** — sonraki isteklerde kullanmak için
  yanıt panelinde herhangi bir değere sağ tıkla, adlandırılmış
  env değişkenine kaydet (`{{access_token}}`, vs.). Surface **Kod
  Kopyala** menüsünün yanında.
- **Examples** — kayıtlı isteğe bağlı yanıt snapshot'ları. Examples
  sekmesinden kaydet / görüntüle / yeniden adlandır / sil. Reload
  arası persist. Mock-server (HTTP üzerinden example servis eden Zig
  sidecar) follow-up olarak kuyruğa alındı.
- **Pre / post-request script'leri** — Postman uyumlu `pm.*` API
  subset'li QuickJS sandbox (`pm.test`, `pm.expect`,
  `pm.environment.set`, `pm.response`, vs.). Pre-script gönderme
  öncesi isteği mutate eder; post-script yanıt üzerinde çalışır.
  Scripts paneli pass/fail tally + console output'u inline gösterir.
  Sandbox limit'leri: 5 sn CPU, 10 MB heap, `fetch` / `XHR` yok.
- **Kod olarak kopyala** — yanıt üstündeki dropdown 6 formatter'la
  (curl, fetch, axios, Python requests, Go net/http, JS XMLHttpRequest).
  Aktif isteği tam çözümlenmiş render eder (env vars genişletilmiş,
  auth uygulanmış) — paste-ready.
- **cURL paste-detect / import** — URL bara curl komutu yapıştır,
  banner parse + composer doldurmayı önerir. `-X`, `-H`, `-d`,
  `--data`, `-u`, `--insecure`, `-L`, vs. handle eder.

## İptal + observability

- **İstek iptali** — Send butonu in-flight durumda kırmızı `⨯ İptal`
  morph eder. `⌘ +.` (canonical macOS abort) aynı handler'ı fire
  eder. AbortSignal fetch path'inde threaded; native bridge path UI'i
  hemen ready'e döndürür, alttaki curl subprocess için soft-cancel
  semantics dökümante.
- **Aria-live response status** — istek geldiğinde screen reader'lar
  status code değişikliklerini duyurur; high-contrast tema görsel
  alternatif olarak available.
- **Astro-style error overlay** — React tree crash ettiğinde,
  in-app overlay syntax-highlighted blame frame, parsed component
  stack, source excerpts ve tek-tık report-copy yüzeye getirir.
  Production source map'leri runtime'da resolve, böylece paste-able
  stack'ler gerçek function + dosya isimlerini referans eder, minified
  id'leri değil.

## Görünüm + his

- **Tema'lar** — system / light / dark / Tokyo Night / GitHub Light
  / high-contrast. Override edilmedikçe OS appearance takip eder.
  Tema seçici **Ayarlar**'da yaşar (üst bar dişli).
- **Ayarlar modal** — tema, dil, default request davranışı
  (timeout-ms, max-redirects, skip-TLS) ve tam klavye kısayol
  referansı için merkezi yer.
- **Türkçe + İngilizce i18n** — her kullanıcı görünür string `t()`
  üzerinden akar. TR source of truth (`frontend/src/lib/i18n/tr.ts`);
  EN onu aynen mirror eder. Herhangi bir locale bir key kaçırırsa
  TypeScript build'i fail eder. Yeni dil eklemek = tek dictionary
  dosyası + `i18n/index.ts` üç satır.
- **In-app özellik rehberleri** — `?` shortcut veya üst bar
  help-circle butonu küratörlü walkthrough'lı modal açar (Hızlı
  başlangıç, Environment'lar, gRPC, İstek iptali, Hızlı geçiş,
  Examples, Streaming protokoller, Postman import). title / group
  / body üzerinde canlı arama. Markdown body'ler build time'da
  `import.meta.glob` ile bundle'a iner.
- **In-app changelog** — üst bar clock-history butonu (veya version
  bump sonrası ilk launch'ta auto-trigger) bu zaman çizelgesini
  vertical kart listesi olarak gösterir. lib/markdown.ts subset
  renderer escape-by-default güvenli; bundle'da marked / dompurify
  bağımlılığı yok.
- **Toast bildirimleri** — kaydet / iptal / kopyala / tema değişimi
  için non-modal feedback. Auto-dismiss; stacklenmiş.

## Güvenilirlik + altyapı

- **IndexedDB persistence** — koleksiyonlar, environment'lar,
  geçmiş, sekmeler ve preferences IDB'de yaşar. Eski localStorage
  5 MB limitini kaldırır (multi-MB Postman import + OpenAPI spec
  artık `QuotaExceededError` yemez). localStorage-era snapshot'tan
  upgrade eden user'lar için ilk read'de tek-seferlik şeffaf
  migration. Persist contract `migrate(persisted, fromVersion)`
  callback'leriyle versiyonlu (v1 → v2 → v3 chain).
- **Native dialog'lar değiştirildi** — `window.confirm` / `alert` /
  `prompt` WKWebView'da custom origin altında sessiz no-op; `useConfirm()`
  hook'lu React modal (Dialog + ConfirmDialog + PromptDialog)
  ile değiştirildi, böylece confirmation gereken her code path
  gerçek surface alır.
- **+ Yeni klasör / + Yeni istek inline-rename** — yan menü create
  flow'ları bozuk `prompt()` path'i auto-named draft + inline rename
  mode ile değiştirildi. `↵` kabul, `Esc` iptal.
- **TreeNode cycle guard** — drag-into-folder validation klasörü
  kendi descendant'larından birine bırakmayı reddeder (collection
  tree'de cycle yaratırdı).
- **React #185 hardening** — her render'da fresh empty object/array
  döndüren selector'lar `Object.freeze({})` module-scope sabitlerine
  + `??` fallback'e stabilize edildi. İki ayrı occurrence fix
  edildi (QuickSwitcher, ExamplesPanel).
- **Bundle-size guardrail** — `scripts/check-bundle-size.sh` CI'da
  1300 KB JS raw / 400 KB gz enforce eder. Şu an bundle ~1207 raw
  / 367 gz; warning threshold 1400'e çıkarıldı, eşleşmesi için.
- **Test'ler + CI** — Phase F: http handler için Zig unit test'leri
  (argv kompozisyonu, header parsing, JSON encoding, error
  surfacing); pure utils + store slice action + lib primitive'leri
  için Vitest; her push'ta her ikisini de çalıştıran GitHub
  Actions.
- **Pre-commit hook'lar** — dnpm üzerinden `zig fmt` +
  `prettier --check`. `bash scripts/install-hooks.sh` ile
  idempotent kurulum.

## Build + supply chain

- **`./build.sh`** — önceki instance'ı öldüren, WebKit asset
  cache'i wipe eden ve taze build edilmiş binary'i fırlatan
  one-shot frontend + Zig build. Release-mode, no-run, frontend-
  only, zig-only, force-reset-state için flag'ler.
- **dnpm secure docker npm** — her npm/Vite operasyonu hardened
  OrbStack container içinde çalışır: non-root UID, tüm cap'ler
  drop, custom seccomp profili, read-only rootfs + project mount,
  `noexec`-li tmpfs, pinned npm registry, two-phase install
  (download scripts-off, rebuild offline) ve her install'dan sonra
  auto-audit. Docker socket mount edilmiyor (escape vector yok).
  Malicious postinstall script'leri `~/.ssh`, `~/.aws` veya browser
  profile'larına ulaşamaz.
- **Project-rooted compose** — clone-and-run convenience için repo
  root'ta `docker-compose.yml`; hardened olanı (`docker-compose.node.yml`)
  gitignore'da ve `dnpm setup` tarafından emit edilir.

## Platform notu

- v0.1.0 için **sadece macOS** (`app.zon`'da
  `.platforms = .{ "macos" }`). Linux / Windows build'leri
  follow-up kuyrukta — zero-native'in WebKit hosting'i platform'a
  özgü glue gerektirir.
- **WebKit, Chromium değil.** Bu Apple'ın `WKWebView`'ı (Safari'nin
  kullandığı aynı engine), zero-native'in macOS host'u üzerinden.
  Davranışsal olarak Chromium'dan stricter `localStorage` /
  `Headers` semantics, Chrome devtools / extension yok ve küçük
  bir set WKWebView-spesifik gotcha (CLAUDE.md'de dökümante).
